## Prerequisites

Regardless of your installation method, MangoWC requires a Wayland-capable environment. If you are building from source, ensure you have the following dependencies:

<Callout type="info" title="Build Dependencies">
- `glibc`
- `wayland`
- `wayland-protocols`
- `libinput`
- `libdrm`
- `libxkbcommon`
- `pixman`
- `git`
- `meson`
- `ninja`
- `libdisplay-info`
- `libliftoff`
- `hwdata`
- `seatd`
- `pcre2`
- `xorg-xwayland`
- `libxcb`
</Callout>

## Install Guide

Choose your distribution below for specific installation instructions.

<Tabs items={['Arch Linux', 'Fedora', 'Gentoo', 'Guix System', 'NixOS', 'PikaOS', 'Manual Build']}>

<Tab value="Arch Linux">
  MangoWC is available in the **Arch User Repository (AUR)**.

  You can install it using an AUR helper like `yay` or `paru`:

  ```bash
  yay -S mangowc-git
  ```

<Callout type="tip">
This package pulls the latest git version, ensuring you have the newest features and fixes.
</Callout>
</Tab>

<Tab value="Fedora">
The package is in the third-party **Terra repository**. First, add the Terra Repository.

<Callout type="warning" title="Requires sudo privileges">
Both commands require root privileges. Use `sudo` if needed.
</Callout>

```bash
dnf install --nogpgcheck --repofrompath 'terra,https://repos.fyralabs.com/terra$releasever' terra-release
```

Then, install the package:

```bash
dnf install mangowc
```
</Tab>

<Tab value="Gentoo">
The package is hosted in the community-maintained **GURU** repository.

<Steps>
<Step>
**Add the GURU repository**
```bash
emerge --ask --verbose eselect-repository
eselect repository enable guru
emerge --sync guru
```
</Step>
<Step>
**Unmask packages**
Add the required packages to your `package.accept_keywords` file:
- `gui-libs/scenefx`
- `gui-wm/mangowc`
</Step>
<Step>
**Install MangoWC**
```bash
emerge --ask --verbose gui-wm/mangowc
```
</Step>
</Steps>
</Tab>

<Tab value="Guix System">
The package definition is described in the source repository.

<Steps>
<Step>
**Add mango channel**
Add to `$HOME/.config/guix/channels.scm`:
```scheme
(cons (channel
        (name 'mangowc)
        (url "https://github.com/mangowm/mango.git")
        (branch "main"))
      %default-channels)
```
</Step>
<Step>
**Install**
After running `guix pull`, you can install MangoWC:
```bash
guix install mangowc
```

Or add it to your system configuration using the mangowc module:
```scheme
(use-modules (mangowc))

(packages (cons*
            mangowc-git
            ... ;; Other packages
            %base-packages))
```
</Step>
</Steps>

<Callout type="tip">
For more information, see the [Guix System documentation](https://guix.gnu.org/manual/devel/en/html_node/Channels.html).
</Callout>
</Tab>

<Tab value="NixOS">
The repository provides a Flake with a NixOS module.

<Steps>
<Step>
**1. Add flake input**

```nix
# flake.nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
    mangowc = {
      url = "github:mangowm/mango";
      inputs.nixpkgs.follows = "nixpkgs";
    };
    # other inputs ...
  };
}
```
</Step>
<Step>
**2. Import the NixOS module**

<Accordions>
<Accordion title="Option 1: Import in configuration.nix">

```nix
# configuration.nix (or any other file that you import)
{inputs, ...}: {
  imports = [
    inputs.mangowc.nixosModules.default
    # .. other imports ...
  ];

  # ...
}
```

</Accordion>
<Accordion title="Option 2: Import directly in flake">

```nix
# flake.nix
{
  # ...

  outputs = { self, nixpkgs, mangowc, ...}@inputs: let  # <- if `mangowc` is not added here you may refer to it using `inputs.mangowc`
    inherit (nixpkgs) lib;
    # ...
  in {
    nixosConfigurations.YourHostName = lib.nixosSystem {
      modules = [
        mangowc.nixosModules.default # or inputs.mangowc.nixosModules.default
        # other imports ...
      ];
    };
  }
}
```

</Accordion>
</Accordions>
</Step>
<Step>
**3. Enable the module**

```nix
# configuration.nix (or any other file that you import)
{
  programs.mango.enable = true;
}
```
</Step>
<Step>
**Extra options**

- `programs.mango.package` - the mango package to use, allows usage of custom mango drvs
- `programs.mango.addLoginEntry` (default: true) - adds login entry to the display manager
</Step>
</Steps>
</Tab>

<Tab value="PikaOS">
MangoWC is available in the **PikaOS package repository**.

You can install it using the `pikman` package manager:

```bash
pikman install mangowc
```
</Tab>

<Tab value="Manual Build">
If your distribution isn't listed, you can build MangoWC from source. You will need to build `wlroots` and `scenefx` manually as well.

<Steps>
<Step>
**Build wlroots**
Clone and install the specific version required (check README for latest version).
```bash
git clone -b 0.19.2 https://gitlab.freedesktop.org/wlroots/wlroots.git
cd wlroots
meson build -Dprefix=/usr
sudo ninja -C build install
```
</Step>
<Step>
**Build scenefx**
This library handles the visual effects.
```bash
git clone -b 0.4.1 https://github.com/wlrfx/scenefx.git
cd scenefx
meson build -Dprefix=/usr
sudo ninja -C build install
```
</Step>
<Step>
**Build MangoWC**
Finally, compile the compositor itself.
```bash
git clone https://github.com/mangowm/mango.git
cd mangowc
meson build -Dprefix=/usr
sudo ninja -C build install
```
</Step>
</Steps>
</Tab>

</Tabs>
